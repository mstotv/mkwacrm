# نظرة عامة ودليل نظام WaCRM (MitaKurd WhatsApp Auto)

هذا الملف هو المرجع الأساسي والشامل لمنصة **WaCRM**؛ حيث يوثق البنية البرمجية، وقواعد البيانات، ونظام الترجمة، وتنسيق الألوان، والصفحات الأساسية. قراءة هذا الملف تغنيك عن تحليل جميع ملفات المشروع عند البدء بأي عمل جديد.

---

## 1. نظرة عامة على المنصة (Platform Overview)
المنصة عبارة عن نظام **إدارة علاقات عملاء (CRM) متعدد المشتركين (SaaS)** مخصص لواتساب، يتيح للشركات والفرق:
* استقبال وإرسال رسائل واتساب عبر صندوق وارد مشترك (Shared Inbox).
* تصنيف جهات الاتصال وتقسيمهم (Contacts Segmentation) ومزامنتها مع Google Sheets.
* إدارة مراحل المبيعات والعملاء (Sales Pipelines).
* إطلاق حملات البث الجماعي (Broadcasts) المجدولة والتلقائية.
* بناء ردود تلقائية ذكية باستخدام الذكاء الاصطناعي (OpenAI & DeepSeek) وبوتات الكلمات المفتاحية (Keyword Bots).
* إنشاء مسارات تفاعلية متقدمة (Flows) بدون كود.

---

## 2. التقنيات المستخدمة (Technology Stack)
* **الموقع والواجهة الخلفية**: Next.js 16.2.6 (App Router) مع React 19.
* **قاعدة البيانات والخدمات السحابية**: Supabase (Database, Auth, Storage, Realtime).
* **التنسيق والتصميم**: Tailwind CSS v4 مع Vanilla CSS وتنسيقات مخصصة (Custom Themes).
* **المكتبات الرسومية وتكامل البيانات**:
  * `@xyflow/react` لبناء مخططات الأتمتة والمسارات.
  * `@dnd-kit/core` لسحب وإفلات العناصر (Drag & Drop).
  * `lucide-react` للأيقونات.
  * `sonner` للتنبيهات (Toasts).
  * `recharts` للرسوم البيانية والإحصائيات.

## 4. بنية قاعدة البيانات (Database Schema Overview)

تدار الجداول والصلاحيات عبر Supabase RLS (Row Level Security)، وتتكون البنية من جداول رئيسية أهمها:
1. **`accounts`**: يمثل الحسابات أو مساحات العمل المشتركة (Tenants).
2. **`profiles`**: يمثل المستخدمين وصلاحياتهم داخل المنصة ورتبهم الإدارية (`platform_role`).
3. **`account_members`**: يحدد علاقة الأعضاء بالحسابات والأدوار الممنوحة لهم (`owner`, `admin`, `agent`, `viewer`).
4. **`invitations`**: إدارة دعوة الموظفين للانضمام إلى مساحة العمل.
5. **`contacts`**: تخزين بيانات العملاء وجهات الاتصال، ومستوى نقاطهم، ولون التصنيف وعنوانهم.
6. **`pipelines` و `pipeline_stages` و `deals`**: إدارة مراحل المبيعات والصفقات.
7. **`broadcasts` و `broadcast_recipients`**: إدارة البث الجماعي وسجلات الاستلام والـ wamid الخاص بالرسائل.
8. **`automations`**: لتخزين بوتات الرد الآلي وقواعد الأتمتة المخصصة.
9. **`flows`**: مسارات تدفق الرسائل التفاعلية (Interactive Flow Chart JSON).
10. **`tickets` و `ticket_messages`**: نظام تذاكر الدعم الفني بين المشترك والإدارة العليا.
11. **`site_settings`**: الإعدادات العامة للمنصة مثل الاسم، الشعار، والألوان الرئيسية.
12. **`landing_page_settings`**: إعدادات الصفحة الهبوطية والأسعار والخطط وقوالب المميزات.

## 5. نظام اللغات والترجمة (Bilingual System - i18n)
* **المسؤول**: الخطاف المخصص `useLanguage` والسياق `LanguageProvider` في المجلد `src/hooks/use-language.tsx`.
* **الملفات**: `ar.json` (العربية) و `en.json` (الإنجليزية) تحت `src/locales/`.
* **طريقة العمل**:
  * يحفظ الاختيار في `localStorage` بمفتاح `app_language`.
  * يتم تطبيق `lang="ar"` أو `lang="en"` وتحديث الاتجاه `dir="rtl"` أو `dir="ltr"` ديناميكياً على عنصر `<html>`.
  * تستخدم دالة الترجمة `t('key.subkey')` لقراءة المفاتيح المتداخلة.
* **صفحة الهبوط (Landing Page)**:
  * تمت ترقيتها لترتبط مباشرة بـ `useLanguage` لتكون متزامنة تماماً مع النظام، مما يتيح التبديل الفوري وتعديل الاتجاه وسلاسة الانتقال عند تسجيل الدخول.

## 6. نظام المظهر والسمات (Theme & Color Modes)
* **المسؤول**: الخطاف `useTheme` والسياق `ThemeProvider` في المجلد `src/hooks/use-theme.tsx`.
* **طريقة العمل**:
  * يتم تخزين وضع المظهر (Dark / Light) في `localStorage` بمفتاح `wacrm.colorMode`.
  * يُطبق المظهر على `<html>` بإضافة الكلاس المناسب (`dark` أو `light`) والخاصية `data-mode`.
  * تعتمد مكونات لوحة التحكم بالكامل على نظام Tailwind الداكن/الفاتح.
  * **صفحة الهبوط (Landing Page)**: أصبحت تدعم الوضعين (Dark & Light) باحترافية، حيث يتم تبديل المتغيرات اللونية الخاصة بها (`--ld-bg`, `--ld-text`, `--ld-border`, الخ) تلقائياً عند تغيير الكلاس المطبق على `<html>` وتحديث الأيقونات وعناصر التنقل بشكل تفاعلي.

## 7. الدليل السريع للمطورين (Quick Development Guide)

### لإضافة مفتاح ترجمة جديد:
1. افتح `src/locales/en.json` وأضف المفتاح باللغة الإنجليزية.
2. افتح `src/locales/ar.json` وأضف نفس المفتاح باللغة العربية متطابقاً تماماً مع الهيكل الإنجليزي.
3. استخدمه في المكونات باستدعاء:
   ```tsx
   const { t } = useLanguage();
   // الاستخدام:
   t('common.save')
   ```

### لتفعيل ميزات الداكن/فاتح في صفحة الهبوط:
* لا تستخدم ألواناً صلبة مثل `color: #0f172a` أو الخلفية البيضاء الثابتة.
* استخدم المتغيرات المخصصة لصفحة الهبوط مثل:
  * الخلفية: `var(--ld-bg)`
  * خلفية مخففة: `var(--ld-bg-subtle)`
  * البطاقات: `var(--ld-card)`
  * لون النصوص الرئيسي: `var(--ld-text)`
  * لون النصوص المساعد: `var(--ld-text-muted)`
  * الحدود والإطارات: `var(--ld-border)`

## 8. سجل الإصلاحات البرمجية والميزات الإضافية (Fixes & Features Log)

تم توثيق وحل المشاكل البرمجية وإضافة الميزات التالية للحفاظ على استقرار النظام وسرعة البناء (Build) وتجربة المستخدم:

### أ. إصلاحات لوحة تحكم الإدارة (`src/app/admin/page.tsx`):
* **مشكلة التجزئة JSX**: تسبب نسيان إغلاق الوسم `<AdminWrapper>` في خطأ في البناء. تم حلها بإضافة الوسم المغلق `</AdminWrapper>` في نهاية حاوية العرض.
* **دوال غير معرّفة**: كانت دالة `createClient` مستخدمة دون استيرادها. تم حلها بإضافة استيراد `createClient` من مكتبة Supabase للعميل.

### ب. إصلاحات حاوية حماية الإدارة (`src/components/admin/admin-wrapper.tsx`):
* **خطأ علامات الاقتباس (Unterminated String)**: السطر الأول كان يحتوي على `"use client';` (خلط بين علامة اقتباس مزدوجة وفردية). تم تعديلها لتصبح `'use client';`.
* **مطابقة أنواع TypeScript**: واجه المترجم مشكلة عند إسناد بيانات المستخدم إلى الحالة الافتراضية `useState(null)`. تم تعريفها كـ `useState<any>(null)` لمنح المرونة وتخطي مشكلة التعيين.

### ج. ميزة شريط العلامات التجارية المتكرر (Infinite Logo Marquee):
* تم بناء مصفوفة تحتوي على أسماء الجهات والعملات الأجنبية المطلوبة (`Meta, Evolution API, WhatsApp, Supabase, ChatGPT, DeepSeek, Google, Google Sheets, Google Calendar, Plisio, Bitcoin, USDT`) لتمريرها ديناميكياً.
* تصميم حركة دوران لا نهائية بالـ CSS تلتزم بالاتجاه الصحيح للنص بناءً على لغة الصفحة (RTL تدور لليمين، و LTR تدور لليسار) مع تأثير تدرج لوني ناعم يتلاشى عند حواف المكون ليتوافق مع الخلفية في الوضعين الفاتح والداكن.

### د. نظام دورات الفوترة المرنة والتجارب المجانية (Flexible Billing Cycles & Free Trials):
* **توسيع البنية الهيكلية:** تم استبدال خيارات الفوترة الثنائية (شهري/سنوي) بمصفوفة خيارات الفوترة `billing_options` في جدول `subscription_plans` لدعم خيارات متعددة مرنة: عدد أيام مخصص، شهري، سنوي، ومدى الحياة (Lifetime) دفعة واحدة بدون تجديد أو تاريخ انتهاء صلاحية.
* **فترات التجربة المجانية (Free Trials):** تفعيل حقل `trial_period_days` في الخطط ومزامنة الرصيد الاستهلاكي للتجربة بحيث يمنع النظام تكرار الحصول عليها للحساب الواحد.
* **واجهات التفعيل والدفع:** تعديل لوحة المسؤول (Admin Panel) لإضافة التفعيل اليدوي متعدد الدورات، وربط بوابات دفع Plisio بآلية الفوترة والاحتساب الجديدة.
* **شريط التنبيه الترويجي:** تم وضع شريط ترويجي وتنبيهي مميز في الشريط الجانبي للمشتركين الذين يخوضون فترات تجريبية يعلمهم بالأيام المتبقية ويوجههم لتنشيط الحساب والدفع.

### هـ. مسار الأسئلة والأجوبة المبرمج (Scripted Q&A Flow / User Input Flow):
* **نوع خطوة مخصص:** إضافة إجراء جديد للسيناريوهات باسم `question_sequence` يسمح للمستخدم بكتابة تسلسل أسئلة يطرحها البوت على العميل خطوة بخطوة وحفظ الردود في متغيرات مستقلة بالذاكرة التخزينية.
* **جدول تتبع الجلسات:** استخدام جدول `automation_qna_sessions` لتسجيل حالة العميل المعلقة، رقم السؤال الحالي، الإجابات المجمعة، وموضع الخطوة القادمة لاستئناف العمل دون انقطاع.
* **اعتراض رسائل العملاء (Webhooks):** دمج دالة المعالجة `handleQnaSessionResponse` في ويب هوكس Meta و Evolution لتمرير الرسائل المستلمة كإجابة فورية وتجميد المحركات المتعارضة (AI/Keywords).
* **الاستئناف التلقائي:** فور تجميع آخر إجابة، تستأنف الأتمتة عملها من النقطة المخزنة (مثل خطوة ترحيل البيانات إلى Google Sheets التي تطابق المتغيرات وتحفظها كصف كامل).
* **محرر الواجهة المرئي:** إضافة محرك تعديل كامل الخطوات في الـ Workflow Builder يتيح إدراج الأسئلة وتسمية حقول المتغيرات وتحديد نوع الإدخال (نص، رقم، ميديا).

### ز. إصلاح نظام الفوترة والاشتراكات اليدوية وتفعيل التجربة المجانية (Subscription RLS & Admin Manual Upgrades Hotfix):
* **مشكلة قراءة الاشتراكات (RLS Select Block):** تم حل مشكلة ظهور الباقات المرقّاة كـ "Free / Inactive" للمستخدمين بسبب سياسات RLS على جدول `account_subscriptions` التي تمنع الـ Client-side query. تم إنشاء مسار API خلفي آمن `/api/billing/subscription` يستعلم بـ `service_role` ويتحقق من JWT لجلسة العميل لخدمة شريط التنقل والفوترة وإعدادات الذكاء الاصطناعي وصوت نجاح الدفع [success/page.tsx](file:///c:/Users/Mustafa/Desktop/wacrm/src/app/billing/success/page.tsx).
* **مشكلة الترقية اليدوية (RLS Write Block):** تم إصلاح شاشات الترقية اليدوية بالأدمن بانل (في صفحة الحسابات وصفحة الاشتراكات) لتتم الكتابة ويسجل تاريخ الدفع `payment_history` عبر طلبات API آمنة موحدة بدلاً من كتابة Supabase مباشرة من المتصفح التي كانت تفشل بصمت بسبب RLS.
* **مشكلة تفعيل الفترة التجريبية المجانية (Trial RLS Block):** تم تعديل مسار تفعيل التجربة المجانية `/api/billing/trial/activate` ليكتب التحديثات في جدول الاشتراكات باستخدام الـ `service_role` بعد التحقق من أحقية المالك والـ JWT بدلاً من طلبات الـ client-side التي كانت تفشل بصمت.
* **إصلاح تحذيرات CSS الكاذبة:** تم إنشاء إعدادات لمنع التحذيرات الكاذبة لـ Tailwind CSS v4 في VS Code عبر `.vscode/settings.json`.
