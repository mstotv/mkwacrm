## 💎 الميزات والحلول المنجزة بالتفصيل | Feature Highlights

### 1. حل مشكلة ظهور الخطط في لوحة المستخدم (`/settings?tab=billing`)
* **السبب السابق**: سياسة RLS لجدول `subscription_plans` كانت تحجب قراءة البيانات للمستخدمين العاديين، مما يجعل واجهة الاشتراك تظهر خالية.
* **التعديل**: تم تحسين سياسة الجدول لتسمح بالوصول العام لجميع المستخدمين الموثقين لقراءة خطط الأسعار والاشتراك والدفع بالعملات الرقمية أو تليجرام بسهولة.

### 2. حل مشكلة إعدادات وتدريب الذكاء الاصطناعي (`/settings?tab=ai`)
* **التعديل على RLS الجدول**: تم تصحيح سياسة الإدخال لجدول `ai_config` للسماح لمالكي الحسابات (`owner` / `admin`) بإضافة وتعديل إعدادات البوت والتدريب بسلاسة دون أخطاء تعارض المفاتيح الفريدة.
* **دعم الفحص والتدريب بشكل اختياري**:
  * تمت إضافة لوحة توجيه ذكية داخل واجهة الـ AI تنبه العميل إذا كانت باقته الحالية (مثل Free أو Starter) لا تدعم ميزة الرد بالذكاء الاصطناعي وتوجهه لقسم الفوترة للترقية لباقة **Pro**، مع الحفاظ على إمكانية حفظ إعداداته وتجربتها بشكل تجريبي.
  * تحديث كود الـ Auto-responder في الخلفية ليتحقق من أن باقة الحساب هي **Pro** وأن البوت مفعل بالخيار الاختياري (`is_active = true`) قبل استدعاء OpenAI/DeepSeek، مما يضمن احترام خيارات العميل وحماية استهلاك الـ API Key الخاص به.

### 3. نظام الدعم الفني المتكامل (Support Tickets System)
* **واجهة العميل (`/support`)**: فتح تذاكر جديدة، رفع المرفقات (صور وفيديو وPDF) في الوقت الفعلي، ومراسلة الدعم بشكل حي.
* **لوحة المساعدين والأدمن (`/admin/tickets`)**: لوحة توزيع التذاكر على المساعدين والرد عليها وتغيير حالتها حتى إغلاق التذكرة.

### 4. صفحات لوحة الأدمن المحدثة
* **إدارة المشتركين (`/admin/accounts`)**: حظر/إلغاء حظر المشتركين يدوياً، ترقية المستخدمين لمساعد أدمن، وتعديل الاشتراكات يدوياً.
* **تخصيص اللاندينج بيج والألوان (`/admin/settings`)**: تحكم كامل بنصوص وألوان صفحة الهبوط فورياً لجميع الزوار.
* **تحليلات النمو (`/admin/analytics`)**: إحصائيات الدخل MRR ورسوم بيانية تفاعلية لنمو العملاء والنشاط.

## 🟢 نظام ربط الواتساب المباشر عبر Evolution API (الربط عبر QR Code)

تم دمج وإعداد خيار الربط المباشر لحسابات الواتساب عبر **Evolution API** إلى جانب ربط Meta Cloud API التقليدي. يتيح هذا الخيار للمستخدمين ربط هواتفهم الشخصية مباشرة بمجرد مسح رمز الـ QR Code من لوحة التحكم، مما يمنحهم حرية كاملة دون الحاجة لتوكينات Meta المعقدة.

### 🌟 تفاصيل المزايا المضافة في هذا التحديث:
1. **التبديل المرن بين المنصتين (Meta & Evolution)**:
   * تم تعديل واجهة إعدادات الواتساب في لوحة تحكم العميل لتضم علامات تبويب (Tabs) تمكنه من التنقل وتفعيل خيار الاتصال المطلوب بشكل فوري وتفاعلي.
2. **إنشاء وإدارة الجلسات تلقائياً (Session Lifecycle Management)**:
   * عند إدخال بيانات المثيل (اسم الجلسة، توكن الحماية، رقم الهاتف) والضغط على "حفظ وإعداد الجلسة"، يقوم النظام بالتواصل المباشر مع خادم Evolution لإنشاء مثيل جديد وتوليد كود الـ QR Code وعرضه بشكل فوري على الواجهة.
3. **فحص ومزامنة الاتصال التلقائي (Status Polling)**:
   * تقوم الواجهة بفحص حالة الجلسة بالخلفية كل 5 ثوانٍ بمجرد ظهور كود الـ QR Code. وعند قيام المستخدم بمسح الكود بهاتفه بنجاح، يتم تحديث الحالة فورياً في قاعدة البيانات وعرض علامة الاتصال الخضراء بنجاح.
4. **توجيه الرسائل الصادرة والواردة ذكياً**:
   * تم تعديل مسارات الإرسال (`/api/whatsapp/send` و `auto-responder.ts`) لتتحقق من نوع الاتصال النشط. إذا كان الاتصال هو `evolution` يتم تمرير الرسائل تلقائياً عبر API الخاص بـ Evolution.
   * تم إنشاء مسار ويب هوك مخصص ومؤمن بالكامل لتلقي الرسائل الواردة من خادم Evolution وتحويلها إلى لوحة المحادثات (Inbox) وتشغيل الأتمتة والمجيب التلقائي بالذكاء الاصطناعي للمستخدمين المشتركين بباقة **Pro**.
